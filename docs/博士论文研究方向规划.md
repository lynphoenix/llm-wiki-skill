## 一、领域现状概览

### 1.1 技术演进路线

```
传统统计方法 (PCA/ISF)
    ↓
深度学习 (LSTM-AE, DAE)
    ↓
图神经网络 (GCN, GAT, Graph Autoencoder)
    ↓
时空Transformer (Informer, PatchTST)
    ↓
扩散模型 + 图 (Diffusion + GNN) ← 2025新热点
    ↓
Foundation Models + LLM (Zero-shot TSAD) ← 2025-2026前沿
```

### 1.2 当前 SOTA 代表工作 (2025-2026)

| 方向 | 代表论文 | 年份 | 核心创新 |
|------|----------|------|----------|
| 扩散模型 | DDTAD, DSCAD, DiffusionAD | 2025 | 用扩散模型生成正常样本，异常重建误差大 |
| 因果解耦 | CDRL4AD | 2025 | 因果 disentanglement + 时序异构图 |
| 概念漂移 | DADA | 2025 | 动态漂移感知 + 扩散增强 |
| LLM/Foundation | AnomLLM (NeurIPS 2025) | 2025-26 | 零样本异常检测，LLM做时序理解 |
| 图级别异常 | MoEGAD (IEEE TPAMI) | 2025-26 | 伪异常生成 + 专家混合，零样本泛化 |
| 增量学习 | I-VGATEPi (IEEE TCYB) | 2025 | 变分推断 + 概率解释 + 无需重训练 |

### 1.3 现有方法的共性局限

- **99%的论文只在模拟数据集验证**（TEP、工业基准），忽略真实工业环境的复杂性
- **忽视概念漂移**：设备老化、季节变化、负载波动导致数据分布变化
- **单模态为主**：只考虑传感器时序，忽略工业现场的多模态信号（振动、声纹、红外）
- **缺乏领域知识整合**：纯数据驱动，可解释性差
- **验证规模小**：方法论创新有限，主要在小规模公开数据集上刷榜

### 方向一：真实工业数据的概念漂移研究 ⭐⭐⭐⭐⭐

#### 研究成熟度

| 维度 | 状态 | 说明 |
|------|------|------|
| 论文数量 | 多 | 2014年 Gama 提出框架后大量 follow-up |
| 综述文章 | 成熟 | 多篇系统性综述 |
| 方法论 | 相对成熟 | 完整的方法分类体系（突变/渐变/增量/周期性） |
| **工业真实数据验证** | **极不成熟** | **你的机会** |

#### 方法体系成熟度

| 漂移类型 | 检测方法 | 成熟度 |
|----------|----------|--------|
| 突变 (Sudden) | DDM, ADWIN, Page-Hinkley | ⭐⭐⭐⭐ 成熟 |
| 渐变 (Gradual) | EDDM, RDDM | ⭐⭐⭐⭐ 较成熟 |
| 增量 (Incremental) | 统计假设检验, DAWIDD | ⭐⭐⭐ 中等 |
| 周期性 (Recurring) | ADWIN, 概念记忆库 | ⭐⭐ 较初步 |
| 混合类型 | 统一框架 | ⭐ 不成熟 |

#### 代表工作及局限性

| 论文 | 方法 | 局限性 |
|------|------|--------|
| DADA (arXiv 2025) | 自适应异常检测 | 只在合成数据验证 |
| CDAM (IEEE TKDE 2023) | Transformer动态调整学习率 | 漂移类型单一 |
| METER (VLDB 2024) | 在线异常检测框架 | 只做检测，不做长期适应 |
| SAPDAD (arXiv 2025) | 实时自适应LSTM | 合成数据验证 |
| ADWIN/Page-Hinkley | 统计漂移检测 | 假设过强，工业场景不适配 |

#### 关键发现

- 方法论研究相对成熟，但几乎**全部在合成数据集上验证**
- 真实工业场景的**长期运行验证（数月）极少数**
- **异常 vs 漂移区分**几乎无人研究——这是工业实际最大痛点但没人解决
- 公开的**真实工业概念漂移数据集几乎空白**

#### 你的机会

```
现有研究: 合成数据 + 方法论创新
你的优势: 真实数据 + 真实问题

差距 = 机会
```

**可切入的具体问题：**
1. 做一个**真实工业过程概念漂移数据集**，标注漂移类型和周期
2. 在你的数据上跑现有方法，证明他们实际表现很差
3. 研究"异常 vs 漂移区分"——这是工业最大痛点但没人研究
4. 研究增量适应策略（不重训练、不遗忘）

#### 风险评估

| 风险 | 级别 | 应对 |
|------|------|------|
| 方法论难突破 | 低 | 不需要颠覆检测方法，需要真实验证 + 问题定义 |
| 数据采集周期长 | 中 | 越早开始越好，3-6个月可观察足够漂移 |
| 同行竞争 | 低 | 做得好的人少 |
| 成果可发表性 | 低 | 真实数据稀缺，高水平期刊愿意收 |

#### 第一步行动

现在就能做：
1. **选定采集设备/产线**（2-3种不同工况）
2. **设计采集方案**：连续采集3-6个月，标注关键事件（换班、换料、设备维护）
3. **观察分布变化**：用简单统计（均值、方差、分布距离）初步看是否存在漂移

### 方向二：工业多模态融合异常检测 ⭐⭐⭐⭐

#### 研究成熟度

| 维度 | 状态 | 说明 |
|------|------|------|
| 论文数量 | 中等 | 近年快速增长 |
| 方法多样性 | 高 | 融合策略多种多样 |
| **真实工业多模态** | **较不成熟** | **你的机会** |

#### 主流方法分类

| 融合层级 | 方法 | 代表工作 |
|----------|------|---------|
| **数据层融合** | 原始信号拼接 | 较少 |
| **特征层融合** | AE/VAE特征拼接 + Attention | MFGAN (Sensors 2024) |
| **决策层融合** | 各模态独立判断后融合 | 多数方法 |
| **混合融合** | 多层级联合优化 | M3DM (CVPR 2023) |

#### 代表工作

| 论文 | 模态组合 | 年份 | 核心创新 |
|------|----------|------|----------|
| **MFGAN** (Sensors) | DCS时序 + 声学信号 | 2024 | Attention+GAN融合，F1比TimesNet高5.6% |
| **M3DM** (CVPR 2023) | 点云 + RGB图像 | 2023 | 混合融合 + Memory Bank |
| **Crossmodal** (CVPR 2024) | 2D图像 + 3D点云 | 2024 | 跨模态特征映射 |
| **Audio+Vibration** | 声学 + 振动 | 2024 | 制造过程异常检测 |
| **多模态实时系统** | 视频 + 音频 | 2025 | 实时异常检测系统 |

#### 关键发现

- **MFGAN (Sensors 2024)** 是目前最接近工业多模态的工作：DCS传感器时序 + 声学信号，F1比单模态高5.6%
- 大部分多模态工作集中在**视觉+点云**（MVTec 3D-AD数据集），**工业时序+其他模态**工作很少
- 融合策略无共识：有人用Attention、有人用GAN、有人用Memory Bank
- **模态选择问题**：不是越多越好，如何选择最优模态组合？

#### 你的机会

工厂天然多模态：
- 传感器时序（温度、压力、流量）
- 振动信号
- 声学/声纹
- 红外图像
- 视觉检测

**你能做的：**
1. 建立**工业多模态异常检测数据集**（传感器 + 你有的其他模态）
2. 研究**异构模态融合**策略
3. 研究**模态缺失/噪声处理**
4. 研究**模态选择**——如何平衡精度和成本

#### 风险评估

| 风险 | 级别 | 应对 |
|------|------|------|
| 跨模态对齐复杂 | 中 | 从两种模态开始，逐步增加 |
| 硬件采集成本 | 中 | 选择已有的传感器组合 |
| 方法论创新空间 | 中 | 融合策略还有很多探索空间 |

#### 你的优势

你有现场设备可以**同时采集多种模态**，而大部分研究者只能访问单一模态数据。

### 方向三：物理信息扩散模型 ⭐⭐⭐⭐

#### 研究成熟度

| 维度 | 状态 | 说明 |
|------|------|------|
| 论文数量 | 快速增长 | 2024-2025爆发 |
| 工业应用 | **初步** | 刚进入工业异常检测 |
| **物理约束整合** | **探索中** | 最佳切入点 |

#### 代表工作

| 论文 | 年份 | 核心创新 |
|------|------|----------|
| **PhysDiff** (NeurIPS 2025) | 2025 | 物理引导信号分解 + 扩散重建 |
| **PIDM** (ICLR 2025) | 2025 | 物理信息扩散模型框架 |
| **Physics-Informed Diffusion** (arXiv) | 2025 | 无监督异常检测 + 物理约束 |
| **PINN for RO membranes** | 2025 | 物理信息神经网络用于膜降解诊断 |

#### 核心方法

**扩散模型用于异常检测的逻辑：**
```
训练：学习"正常样本"的生成分布
检测：异常点无法被有效重建 → 重建误差大
```

**物理信息扩散的创新点：**
```
传统扩散：纯数据学习正常分布
物理信息扩散：扩散过程受物理方程约束
         → 正常样本 = 物理合理的数据
         → 异常 = 违反物理约束
```

#### 关键论文解析

**PhysDiff (NeurIPS 2025)——最新最相关的工作：**

核心三部分：
1. **物理引导信号分解**：分离高频振荡和低频趋势，减少干扰
2. **振幅敏感排列熵准则**：自适应确定最优分解深度
3. **双路径条件扩散网络**：高频/低频物理先验通过时频能量路由机制融合

**PIDM (ICLR 2025)：**
- 物理信息作为扩散模型的约束条件
- 生成样本必须满足物理方程
- 在多个物理系统上验证

#### 你的机会

你已有 **physics-informed GRAT**，向扩散模型延伸是**自然路径**：

```
你的积累 → 扩展方向
physics-informed GRAT → + 扩散生成 = PhysDiff式方法
```

**可切入的具体问题：**
1. 工业过程的**物理方程**如何融入扩散模型？
2. 如何用扩散模型建模"正常工况"的生成空间？
3. 异常检测 = 物理约束违反 + 统计异常，**如何联合评分**？
4. **推理速度**：扩散模型慢，如何加速到实时？

#### 风险评估

| 风险 | 级别 | 应对 |
|------|------|------|
| 扩散模型实现复杂 | 中 | 有开源代码可参考 |
| 物理约束获取难 | 中 | 从简化的能量/物料平衡方程开始 |
| 推理速度慢 | 中 | 知识蒸馏、一步扩散等加速方法 |

### 方向四：时序基础模型（长期，高风险高回报） ⭐⭐⭐

#### 研究成熟度

| 维度 | 状态 | 说明 |
|------|------|------|
| 基础模型 |快速发展 | Chronos, Lag-Llama, TimesFM, Moirai |
| 零样本能力 | 部分验证 | 在预测任务上好，异常检测差 |
| **工业领域适配** | **极不成熟** | **你的机会** |

#### 基础模型生态

| 模型 | 机构 | 特点 |
|------|------|------|
| **Chronos** | Amazon | 将LLM范式迁移到时序，tokenize时序为token |
| **Lag-Llama** | 多个机构 | 首个开源时序基础模型，decoder-only Transformer |
| **TimesFM** | Google | 预测任务强，异常检测弱 |
| **Moirai** | Salesforce | Universal transformer，多任务 |
| **AnomLLM** (NeurIPS 2025) | - | LLM用于时序异常检测，发现LLM比简单方法还差 |
| **TimeRCD** (ICLR 2026) | - | 相对上下文差异预训练，零样本异常检测 |

#### 关键发现

**AnomLLM (NeurIPS 2025) 的重要结论：**
- LLM用于时序异常检测**表现不佳**，甚至不如简单方法
- 挑战1：缺乏对复杂时序动态的理解
- 挑战2：难以精确异常定位
- 提出统计分解 + 索引感知提示，F1提升66.6%

**TimeRCD (ICLR 2026 under review) 的创新：**
- 相对上下文差异（RCD）策略
- 超越重建学习，显式学习识别异常
- 零样本异常检测新范式

#### 你的机会

**现有基础模型的局限：**
- 预训练数据：通用时序，**缺乏工业领域**
- 下游任务：预测好，异常检测差
- 零样本：预测任务可以，异常检测不行

**你能做的：**
```
在真实工业数据上预训练 → 学到工业通用表征
         ↓
迁移到新产线/新设备 → 少样本/零样本异常检测
         ↓
领域自适应 → 通用预训练 + 工业微调
```

这是**护城河最高**的方向，一旦建立数据集和模型，壁垒很深。

#### 风险评估

| 风险 | 级别 | 应对 |
|------|------|------|
| 需要大量算力 | 高 | 寻求校内合作或云服务 |
| 数据规模要求高 | 高 | 联合多产线数据 |
| 竞争激烈 | 中 | 大厂（Amazon, Google）做通用，你做工业垂直 |
| 研发周期长 | 高 | 24个月+，需要长期投入 |

### 方向五：因果驱动的异常根因定位 ⭐⭐⭐

#### 研究成熟度

| 维度 | 状态 | 说明 |
|------|------|------|
| 因果发现算法 | 成熟 | PC、FCI、Granger因果等 |
| 工业根因分析 | **较成熟但缺数据驱动** | 有方法但验证不足 |
| **因果+异常联合** | **初步** | 刚起步 |

#### 核心技术路线

| 方法 | 原理 | 代表工作 |
|------|------|---------|
| **Granger因果** | 时间滞后预测能力判断因果 | 条件Granger因果 + 最大生成树 |
| **PC算法** | 条件独立性检验发现因果图 | 因果发现 + 故障诊断 |
| **因果发现+优化** | 联合优化因果结构和模型 | 统一框架 (IFAC 2024) |
| **因果解耦** | VAE解耦因果和相关信息 | CDRL4AD (arXiv 2025) |
| **因果推断+RCA** | 多模态因果推断 | 大型复杂系统因果RCA (2024) |

#### 代表工作

| 论文 | 方法 | 局限性 |
|------|------|--------|
| **条件Granger因果** | 最大生成树根因定位 | 假设线性关系 |
| **统一因果框架** | Granger因果 + 故障诊断联合 | 计算复杂 |
| **CDRL4AD** (arXiv 2025) | 因果解耦 + 时序异构图 | 纯数据驱动，缺领域知识 |
| **多模态因果RCA** | 多模态数据因果推断 | 验证不充分 |
| **图神经网络+RCA** | GNN + 知识图谱 | 需要先验因果知识 |

#### 关键发现

- **工业RCA主流方法**：基于Granger因果的时间序列根因诊断
- **核心问题**：因果发现算法假设强（线性/高斯），工业数据往往不满足
- **最大挑战**：从工艺知识构建的因果图如何与数据驱动发现结合？
- **CDRL4AD**是目前最新的因果+异常检测工作，但纯数据驱动，你的工艺知识可以补充

#### 你的机会

**现有工作的局限：**
- 纯数据驱动的因果发现：假设过强，工业场景不符
- 有方法但**缺乏在真实工业数据上的系统验证**
- 工艺知识未被有效利用

**你能做的：**
1. **半监督因果图构建**：从工艺知识出发，用数据验证和修正
2. **异常+根因联合**：检测到异常后，追踪根因传播路径
